<objective>
Build the owner SMS notification system: outbound alerts for urgent leads, inbound command parsing to update lead status, and daily summary cron.

Purpose: The notification loop is the "secret weapon" — it makes the system feel like a real employee reporting to the owner. Without it, leads are captured but the owner never knows. This plan also wires the notification trigger into the webhook handler from Plan 03.
Output: A complete two-way SMS system where urgent calls trigger immediate owner alerts, owner replies update lead status in real time, and a daily summary cron provides end-of-day reporting.
</objective>

  <behavior>
    Command parser — maps free-form SMS text to structured actions:
    - "call back", "callback", "CALL BACK", "call them", "call them back" → { action: 'contacted' }
    - "booked", "scheduled", "book them", "confirmed" → { action: 'booked' }
    - "not interested", "pass", "lost", "no thanks", "ignore" → { action: 'lost' }
    - "stop", "pause notifications", "too many texts" → { action: 'pause' }
    - "resume", "start", "turn on" → { action: 'resume' }
    - anything else → { action: 'unknown', raw: originalText }
    - Parser is case-insensitive and trims whitespace

    SMS templates:
    - Urgent lead: "URGENT: [Name] called about [service]. Score: 9/10. Call back: [phone]. [summary]"
    - Missed call: "Missed call from [number] at [time]. [summary if available]"
    - Daily summary: "Daily summary: [N] calls, [M] new leads, [K] booked. [Top lead if exists: 'Best lead: [name] ([service], score X/10)']"

    Twilio inbound webhook:
    - Validates Twilio signature (reject 403 if invalid)
    - Finds client by owner_phone
    - Gets most recent notification for context (to know which lead to update)
    - Parses command and updates lead status if recognized
    - Logs owner_response to notifications table
    - Returns TwiML response (plain text "Got it!" or confirmation)
  </behavior>

    ```typescript
    import twilio from 'twilio'
    import { env } from '@/lib/utils/env'

    export function verifyTwilioSignature(signature: string, url: string, params: Record<string, string>): boolean {
      return twilio.validateRequest(env.twilioAuthToken(), signature, url, params)
    }

    export async function sendOwnerSMS(
      payload: NotificationPayload,
      supabase: ReturnType<typeof createServiceClient>
    ): Promise<void> {
      const client = twilio(env.twilioAccountSid(), env.twilioAuthToken())
      const message = formatMessageForType(payload)  // calls appropriate template

      let status: 'sent' | 'failed' = 'sent'
      try {
        await client.messages.create({
          body: message,
          from: env.twilioPhoneNumber(),
          to: payload.recipient_phone,
        })
      } catch (err) {
        status = 'failed'
        console.error('SMS send failed:', err)
      }

      // Always log notification attempt (even failures)
      await supabase.from('notifications').insert({
        client_id: payload.client_id,
        call_id: payload.call_id ?? null,
        lead_id: payload.lead_id ?? null,
        type: payload.type,
        channel: 'sms',
        recipient_phone: payload.recipient_phone,
        message,
        status,
      })
    }
    ```
